GameMode — Oyunun Anayasası

Her UE5 oyununun görünmez bir "kural kitabı" vardır: GameMode. Kim oynuyor, nasıl kontrol ediliyor, ekranda ne görünüyor, kazanma koşulu ne? Bu bölümde oyununun iskeletini kuran bu sistemi öğreniyoruz.

3.1

GameMode nedir, neden var?

Bir oyun düşün: kim oynuyor (karakter), nasıl kontrol ediliyor (klavye/gamepad), ekranda ne görünüyor (can barı, skor), kurallar neler (kaç can, nasıl kazanılır)? Tüm bu soruların cevabını tek bir yerde toplayan yapıya GameMode denir.

GameMode, oyunun beyni değil anayasasıdır. Tek tek ne yapılacağını söylemez; "bu oyunda kurallar şunlar, oyuncu şu, kontrolcü bu" diye çerçeveyi çizer.

Gerçek hayat benzetmesi

GameMode bir spor müsabakasının kural kitabıdır. Futbol mu basketbol mu oynanıyor? Kim hakem, kim oyuncu? Puan nasıl sayılıyor? Maç ne zaman biter? Kural kitabı bunları belirler ama topa kendisi vurmaz. Farklı sahada (level) farklı kurallar istiyorsan, o sahaya farklı bir kural kitabı (GameMode) verirsin.

3.2

GameMode neyi belirler? — 4 temel taş

GameMode dört ana "sınıfı" birbirine bağlar. Her biri oyunun farklı bir parçasından sorumlu:

Default Pawn Class

Oyuncunun "bedeni". Oyuna girdiğinde hangi karaktere/araca sahip olacak? Görünüm, boyut, yetenekler buradan gelir.

= Karakter

Player Controller

Bedeni kim/nasıl kontrol ediyor? Klavye, gamepad, dokunmatik girdilerini alıp Pawn'a iletir. Input'un beyni.

= Kontrol

HUD Class

Ekranda ne görünüyor? Can barı, skor, mühimmat, mini harita... Oyuncunun gördüğü 2D arayüz katmanı.

= Arayüz

Game State

Oyunun genel durumu: kaç oyuncu var, toplam skor, kalan süre. Tüm oyuncuların ortak gördüğü bilgi.

= Genel Durum

Bunlar birbiriyle nasıl bağlanır?

Aşağıdaki şema, GameMode'un bu parçaları nasıl bir araya getirdiğini gösterir:

GameMode Hiyerarşisi Kim kimi yönetir?
3.3

GameMode mu, GameModeBase mi?

Blueprint oluştururken iki seçenek görürsün: GameModeBase ve GameMode. Hangisini seçeceğin kafa karıştırır. İşte net cevap:

ÖzellikGameModeBaseGameMode
Ne zaman? Çoğu oyun için ✓ (önerilen) Match/round bazlı oyunlar
Ağırlık Hafif, sade Ekstra match state mantığı içerir
Match State Yok Var (WaitingToStart, InProgress, WaitingPostMatch...)
Tipik kullanım Single-player, basit oyunlar Çok oyunculu arena, tur bazlı maçlar
Kısa cevap

Yeni başlıyorsan ve emin değilsen: GameModeBase seç. İhtiyacın olduğunda GameMode'a geçersin. Çoğu indie/single-player oyun GameModeBase ile yazılır.

3.4

İlk GameMode'unu oluştur

1
Blueprint GameMode oluştur

Content Browser'da boş alana sağ tık → Blueprint Class. Açılan pencerede "GameModeBase" yaz, filtrele, seç. İsim ver: BP_IlkGameMode.

2
İçini incele

Çift tıkla. Açılan editörde Class Defaults butonuna bas (üst toolbar). Sağdaki Details panelinde "Classes" bölümünü göreceksin: Default Pawn Class, HUD Class, Player Controller Class vb. Şu an boş bırakabilirsin.

3
Global olarak ata

Üst menü Edit → Project Settings. Sol listede "Maps & Modes". "Default GameMode" dropdown'undan BP_IlkGameMode'u seç. Bu, tüm proje için varsayılan GameMode olur.

4
Test et

P ile oyna. Görsel bir fark olmayacak ama artık senin GameMode'un arka planda çalışıyor. Doğrulama için: Window → World Settings aç, "GameMode Override" alanında senin GameMode'unu görmelisin (veya "(Project Setting)" yazısı).

3.5

Global mi, seviye bazlı mı atama?

GameMode'u iki yerden atayabilirsin. Farkı bilmek, çok seviyeli oyunlarda kritik:

Project Settings → Maps & Modes
🌍 Tüm proje için varsayılan
World Settings → GameMode Override
📍 Sadece o seviye için

Örnek senaryo: Oyununda 3 seviye var. İlk ikisi normal oyun (BP_GameMode_Play), üçüncüsü bir bulmaca odası (BP_GameMode_Puzzle). Project Settings'e BP_GameMode_Play'i koyarsın (varsayılan), üçüncü seviyenin World Settings'inde Override olarak BP_GameMode_Puzzle seçersin. Override, o seviye için Project ayarını ezer.

Project Setting (varsayılan)
→
World Settings Override (varsa öncelikli)
Sık yapılan hata

Bir seviyede GameMode çalışmıyorsa, önce World Settings → GameMode Override dolu mu kontrol et. Override doluysa Project Settings'teki ayar görmezden gelinir — yanlış GameMode'u test ediyor olabilirsin.

3.6

Yaygın hatalar ve çözümleri

"Play'e bastım ama karakterim yok / havada uçuyorum"

GameMode'da Default Pawn Class boş veya yanlış. Eğer hiçbir Pawn atamazsan UE5 varsayılan "DefaultPawn"u (uçan bir kamera) kullanır. Kendi karakterini görmek istiyorsan Default Pawn Class'a kendi Character Blueprint'ini ata.

"Karakterim spawn olmuyor / yanlış yerde başlıyor"

Sahnede Player Start aktörü yok demek. Place Actors panelinden "Player Start" ara, sahneye yerleştir. Oyuncu bu noktada doğar. Yoksa sahnenin (0,0,0) noktasında başlar.

"GameMode'u değiştirdim ama hiçbir şey olmadı"

Muhtemelen World Settings Override hâlâ eski GameMode'u gösteriyor. Project Settings'i değiştirmek, Override'ı olan bir seviyeyi etkilemez. World Settings'i kontrol et.

Hap Bilgiler

  • GameMode = kural kitabı. Çerçeveyi çizer, topa kendisi vurmaz.
  • 4 temel taş: Pawn (beden), Controller (kontrol), HUD (arayüz), GameState (durum).
  • Emin değilsen GameModeBase seç. GameMode match-state'li oyunlar için.
  • Controller, Pawn'a "possess" eder. Kontrol bu bağ üzerinden akar.
  • World Settings Override, Project Setting'i ezer. Seviye bazlı GameMode böyle yapılır.
  • Player Start olmadan karakter (0,0,0)'da doğar — sahneye eklemeyi unutma.
  • Default Pawn Class boşsa uçan kamera (DefaultPawn) gelir.

Pratik Projeler

Oyununun iskeletini kur
Kolay~10 dk
Proje 1: İlk GameMode'unu oluştur ve ata

Yukarıdaki adımları takip ederek BP_IlkGameMode oluştur, Project Settings → Maps & Modes'a ata. Play'e bas. World Settings'ten GameMode'unun aktif olduğunu doğrula.

Tamamlama Kontrolü
  • GameModeBase tabanlı BP oluşturuldu
  • Project Settings'e atandı
  • World Settings'te doğrulandı
  • Play çalıştı (hata yok)
GameMode oluşturma ve atama iş akışını öğrenmek.
Orta~25 dk
Proje 2: Karakterli GameMode + Player Start

Bir Third Person template karakteri kullan (veya ekle). GameMode'unda Default Pawn Class'a bu karakteri ata. Sahneye Player Start yerleştir. Play'e bastığında o noktada, kendi karakterinle doğmalısın.

Tamamlama Kontrolü
  • Default Pawn Class karaktere atandı
  • Sahnede Player Start var
  • Play'de doğru yerde, doğru karakterle doğuldu
  • WASD ile hareket çalışıyor
Pawn + Controller + Player Start üçlüsünü bağlamak.
İleri~35 dk
Proje 3: İki seviye, iki GameMode

İki seviye (level) oluştur: L_Oyun ve L_Bulmaca. İki GameMode yaz: BP_GM_Oyun (normal karakter) ve BP_GM_Bulmaca (uçan kamera/DefaultPawn). BP_GM_Oyun'u Project Settings'e koy. L_Bulmaca'nın World Settings'ine Override olarak BP_GM_Bulmaca koy. İki seviyede de Play'leyip farkı gözlemle.

Tamamlama Kontrolü
  • 2 ayrı level oluşturuldu
  • 2 ayrı GameMode yazıldı
  • Project Setting + World Override doğru kuruldu
  • Her seviyede farklı Pawn davranışı görüldü
  • Override'ın Project ayarını ezdiği doğrulandı
Seviye bazlı GameMode mimarisi — gerçek çok-seviyeli oyun yapısı.
