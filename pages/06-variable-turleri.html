Variable Türleri — Veriyi Saklamak

Bir oyun aslında sürekli değişen veriden ibarettir: can, skor, isim, konum, envanter... Bu verileri saklayan kutulara variable (değişken) denir. Bu bölümde her veri tipini rengi, ne sakladığı ve gerçek örneğiyle tek tek tanıyacağız — Array, Set ve Map'i de detaylıca.

6.1

Variable nedir? — Etiketli kutu

Bir variable, içine bir değer koyduğun, etiketli bir kutudur. Kutuya bir isim verirsin (Health), bir tür seçersin (sayı), içine değer koyarsın (100). Sonra istediğin zaman kutuyu açıp değeri okur (Get) veya değiştirirsin (Set).

Gerçek hayat benzetmesi

Variable'lar etiketli kavanozlardır. "Şeker" yazan kavanoza şeker, "Tuz" yazana tuz koyarsın. Tür önemli: "Sıvı" kavanozuna katı koyamazsın. UE5'te de "Sayı" kutusuna metin koyamazsın — her kutunun bir türü vardır ve o türü saklar.

Get ve Set — Okuma ve Yazma

Bir variable'ın iki kullanımı
Get Health
değeri oku
Set Health
exec
yeni değer
exec

Get (sarı, sade) değeri okur — sıraya girmez. Set (mavi, exec'li) değeri değiştirir — sıraya girer çünkü "bir şeyi değiştirmek" bir eylemdir. Variable'ı My Blueprint panelinden graph'a sürüklerken UE5 sana "Get mi Set mi?" diye sorar.

Variable nasıl oluşturulur?

1
My Blueprint → Variables → +

Sol paneldeki Variables başlığının yanındaki + butonuna bas. Yeni bir variable belirir, ismini yaz.

2
Tür seç

Details panelinde Variable Type dropdown'undan tür seç (Boolean, Float, String...). Renk otomatik o türe göre olur.

3
Compile → Varsayılan değer ata

Compile et. Sonra Details'te Default Value alanına başlangıç değerini gir (örn. Health = 100).

6.2

Temel veri türleri — Renk renk

İşte en çok kullanacağın temel türler. Her birinin sabit bir rengi var (pin rengiyle aynı). Rengi tanıyınca türü anlarsın:

Boolean (bool) Kırmızı

Ne saklar: Sadece iki değer — true (doğru) veya false (yanlış). Açık/kapalı, var/yok, evet/hayır.

Örnek: IsAlive (hayatta mı?), HasKey (anahtar var mı?), IsDoorOpen (kapı açık mı?).

Branch (if) node'u boolean ile çalışır — kararların temelidir.
Integer (int) Camgöbeği

Ne saklar: Tam sayı — ondalık yok. 0, 1, 42, -7, 1000. Sayma işleri için.

Örnek: Score (skor), Lives (can sayısı), AmmoCount (mermi), Level (seviye no).

Sayılabilen şeyler (3 can, 5 mermi) için integer; ölçülen şeyler (2.5 metre) için float.
Float Açık yeşil

Ne saklar: Ondalıklı sayı — 3.14, 0.5, 99.9, -12.34. Hassas/ölçülen değerler.

Örnek: Health (can: 87.5), Speed (hız), Damage (hasar), Timer (süre).

Can, hız, mesafe, süre gibi çoğu oyun değeri float'tır. UE5'te varsayılan ondalık türü.
String Pembe

Ne saklar: Metin — harfler, kelimeler, cümleler. "Merhaba", "Oyuncu1", "Level tamamlandı".

Örnek: PlayerName (oyuncu adı), debug mesajları, dosya yolları.

Print String string alır. Oyuncuya gösterilecek kalıcı metinler için Text daha iyidir (çeviri desteği).
Name Lavanta

Ne saklar: Kısa, değişmez etiket/anahtar. String'e benzer ama daha hızlıdır ve karşılaştırma için optimize edilmiştir.

Örnek: Kemik adı ("head"), socket adı, tag, satır adı (DataTable).

Gösterilmeyecek, sadece sistem içi kimlik/anahtar olan metinler Name olmalı (performans).
Text Açık leylak

Ne saklar: Oyuncuya gösterilecek metin. Çeviri (lokalizasyon) destekler — farklı dillere çevrilebilir.

Örnek: UI yazıları, diyaloglar, buton etiketleri, "Oyun Bitti" ekranı.

Ekranda görünecek her şey Text olmalı — ileride oyununu başka dile çevirmek istersen kolaylık sağlar.
6.3

String mi, Name mi, Text mi? — Karışıklığı bitir

Bu üçü yeni başlayanları en çok karıştıran konudur. İşte net ayrım:

TürAmaçÇeviri?HızKullan
NameSistem içi anahtar/kimlikHayırEn hızlıKemik, tag, satır adı
Stringİşlenen/geçici metinHayırOrtaDebug, hesap, birleştirme
TextOyuncuya gösterilen✓ EvetYavaşUI, diyalog, menü
Pratik kural

Ekranda görünecek mi? → Text. Sistem içi kimlik mi? → Name. Geçici işlenen metin mi (debug, birleştirme)? → String. Aralarında dönüşüm node'ları var (ToText, ToString), istediğinde çevirirsin.

6.4

Yapısal türler — Vector, Rotator, Transform

Bazı veriler tek sayı değildir; birden çok sayıyı bir arada tutar. 3D dünyada konum, dönüş ve ölçek bu türdendir:

Vector Sarı

Ne saklar: 3 float bir arada — X, Y, Z. Genelde 3D uzayda bir konum veya yön.

Örnek: Actor konumu (1000, 250, 80), hareket yönü, kuvvet vektörü, ölçek.

Eksen renkleri: X kırmızı, Y yeşil, Z mavi (mavi = yukarı/aşağı).
Rotator Açık mavi

Ne saklar: 3 açı — Pitch (yukarı/aşağı), Yaw (sağ/sol), Roll (yatış). Bir nesnenin dönüşü.

Örnek: Karakterin bakış yönü, dönen kapı, kamera açısı.

Pitch = başını eğme, Yaw = sağa-sola dönme, Roll = yana yatma (uçak gibi).
Transform Turuncu

Ne saklar: 3'ü bir arada — Location (Vector) + Rotation (Rotator) + Scale (Vector). Bir nesnenin tam "duruşu".

Örnek: Spawn noktası (nerede + hangi açıyla + ne boyutta), kayıt edilen poz.

Bir Actor'ı tam olarak konumlandırmak için tek transform yeterli — 3 bilgiyi birden taşır.
Bunlar birer "Struct"tır

Vector, Rotator, Transform aslında Struct (yapı) örnekleridir — yani birden çok değeri paketleyen kaplardır. Pinlerini "Split" ederek (sağ tık → Split Struct Pin) içindeki tek tek X, Y, Z değerlerine ulaşabilirsin. Kendi struct'ını da yaratabilirsin (6.6).

6.5

Referans türleri — Object ve Class

Bazen bir variable bir sayı/metin değil, başka bir nesnenin kendisini tutar. Bunlara referans denir:

Object Reference

Sahnedeki belirli bir nesneyi işaret eder. "Şu kapı", "bu düşman". Mavi pin. O nesnenin fonksiyonlarını çağırabilirsin.

= Belirli bir örnek

Class Reference

Bir nesne türünü işaret eder, örneği değil. "Düşman tipi" (hangi düşmanı spawn edeceğim?). Mor pin.

= Şablon/tür
Object vs Class farkı

Class bir kalıptır — "kurabiye kalıbı". Object o kalıptan çıkan gerçek kurabiyedir. Spawn Actor node'una "hangi kalıbı kullanayım?" diye Class verirsin, o da sana gerçek bir Object (kurabiye) üretir. Class = tür, Object = o türün canlı örneği.

6.6

Koleksiyonlar — Array, Set, Map

Tek bir değer yetmediğinde, birden çok değeri bir variable'da toplarsın. UE5'te 3 koleksiyon türü vardır. Hepsi herhangi bir temel türle çalışır (Integer array, String array vb.). Pin şekilleri farklıdır:

Array (Dizi) İçi boş kareler

Ne saklar: Sıralı liste. Aynı değer tekrar olabilir. Her elemanın bir index'i (sırası) vardır: 0, 1, 2...

Örnek: Envanter [kılıç, kalkan, iksir], yüksek skor listesi, bekleyen düşmanlar.

Eleman sayısı
Index 0'dan başlar! İlk eleman [0], ikinci [1]. 3 elemanlı dizide son index 2'dir.
Set (Küme) Dolu yuvarlak kümeler

Ne saklar: Tekrarsız küme. Aynı değer iki kez olamaz. Sıra garantisi yoktur. "Bir şey içinde var mı?" sorusu çok hızlıdır.

Örnek: Toplanan benzersiz rozetler, ziyaret edilen bölgeler, aktif efektler (her efekt bir kez).

"Bu oyuncu bu görevi tamamladı mı?" gibi benzersizlik/aitlik sorularında Set, Array'den hızlıdır.
Map (Sözlük) Dolu kareler

Ne saklar: Anahtar → Değer eşleşmesi. Her anahtar benzersiz, her anahtarın bir değeri var. Sözlük gibi: kelime → anlam.

Örnek: Oyuncu skorları ("Ali"→90, "Veli"→75), eşya fiyatları ("kılıç"→100), dil tablosu.

"İsme göre değer bul" gerektiğinde Map kullan — anahtarla anında değere ulaşırsın, aramaya gerek yok.

İleri seviye — Doğru koleksiyonu seçmek performanstır

1000 elemanlı bir Array'de "bu var mı?" diye aramak yavaştır (en kötü ihtimalle 1000 kontrol). Aynı işi Set veya Map saniyenin çok küçük bir kısmında yapar (hash tablosu). Büyük veri kümelerinde yanlış koleksiyon seçimi oyununu kasabilir. Kural: sıra gerekiyorsa Array, benzersizlik/aitlik gerekiyorsa Set, anahtar→değer gerekiyorsa Map.

6.7

Kendi türlerin — Struct ve Enum

Hazır türler yetmediğinde kendi türlerini yaratabilirsin:

Struct (Yapı)

Birden çok variable'ı tek pakette toplar. Örn. "ItemData" = İsim (String) + İkon (Texture) + Fiyat (Int) + Ağırlık (Float). Bir eşyanın tüm bilgisi tek variable'da.

Veri paketi

Enum (Seçenek listesi)

Sabit seçenekler kümesi. Örn. "GameState" = {MainMenu, Playing, Paused, GameOver}. Sadece bu değerlerden biri olabilir — yazım hatası riski yok.

Sınırlı seçenek
Ne zaman Struct, ne zaman Enum?

Bir şeyin birden çok özelliği varsa (eşyanın adı, fiyatı, ikonu) → Struct. Bir şey sınırlı sayıda durumdan biri olacaksa (oyun durumu: menü/oynanıyor/duraklatıldı) → Enum. Enum, Branch yerine Switch on Enum ile çok temiz akış sağlar.

İleri seviye — Struct + Array + DataTable

Profesyonel veri yönetiminde bu üçü birleşir: bir Struct tanımlarsın (eşya şablonu), onu bir DataTable'da satır satır doldurursun (tüm eşyalar), oyunda Name anahtarıyla çekersin. Yüzlerce eşyayı/düşmanı/seviyeyi Blueprint'i şişirmeden yönetmenin yolu budur. Inventory, item, dialog sistemleri hep böyle kurulur.

6.8

Göz ikonu — Public/Private ve Instance Editable

Bir variable'ın yanındaki göz ikonuna tıklayınca "açık göz" olur. Bu, variable'ı Instance Editable (public) yapar — sahnedeki her kopyada Details panelinden ayrı ayrı değer girebilirsin.

Kapalı göz (private)
Sadece Blueprint içinden erişilir
Açık göz (public)
Sahnede her kopyada ayarlanabilir

Örnek: Bir "Düşman" Blueprint'i yaptın, Health'i public yaptın. Sahneye 3 düşman koydun. Her birinin Details'inde farklı can girebilirsin: biri 100, biri 250, biri 500. Tek Blueprint, farklı kopyalar.

Tooltip ekle

Public variable'lara Details → Tooltip alanından açıklama ekle. Üzerine gelince ne işe yaradığı görünür — ekip çalışmasında ve aylar sonra kendine hatırlatmada çok değerli.

Hap Bilgiler

  • Bool = doğru/yanlış (kırmızı), Integer = tam sayı (camgöbeği), Float = ondalık (yeşil).
  • Sayılan şey integer, ölçülen şey float.
  • String işle, Name anahtarla, Text göster.
  • Vector = X,Y,Z konum; Rotator = dönüş; Transform = üçü birden.
  • Get okur (sarı, sade), Set değiştirir (mavi, exec'li).
  • Array sıralı+tekrarlı, Set tekrarsız, Map anahtar→değer.
  • Array index 0'dan başlar! İlk eleman [0].
  • Object = belirli örnek, Class = tür/şablon.
  • Struct birden çok özellik, Enum sınırlı seçenek için.
  • Açık göz = public = sahnede her kopyada ayarlanabilir.

Pratik Projeler

Variable'ları gerçek sistemlerde kullan
Kolay~20 dk
Proje 1: Sayaç

Bir Integer variable Counter (varsayılan 0) oluştur. Overlap event'inde Set Counter = Counter + 1 yap, Print String ile "Sayaç: " + Counter göster (ToString ile çevir). Karakter her değdiğinde sayı artsın.

Tamamlama Kontrolü
  • Integer variable oluşturuldu
  • Get + Set ile değer artırıldı
  • Integer → String dönüşümü yapıldı
  • Ekranda artan sayı görüldü
Get/Set döngüsünü ve tür dönüşümünü öğrenmek.
Orta~35 dk
Proje 2: Basit envanter (Array)

Bir String Array Inventory oluştur. 3 farklı toplanabilir eşya Actor'ı yap (Kılıç, Kalkan, İksir). Her biri overlap'te kendi adını oyuncunun Inventory array'ine Add etsin. E tuşuna basınca ForEachLoop ile envanteri ekrana yazdır.

Tamamlama Kontrolü
  • String Array oluşturuldu
  • Add ile eleman ekleniyor
  • ForEachLoop ile liste dolaşılıyor
  • Toplanan tüm eşyalar yazdırılıyor
  • Index/Length kavramı anlaşıldı
Array kullanımı, Add ve ForEachLoop ile koleksiyon işlemek.
İleri~55 dk
Proje 3: Skor tablosu (Map + Struct)

Bir Struct OyuncuBilgi yarat (İsim: String, Skor: Int, Seviye: Int). Bir Map SkorTablosu (anahtar: String isim, değer: OyuncuBilgi struct) oluştur. 3 oyuncu ekle. Bir isme göre Find ile oyuncuyu bul ve bilgilerini ekrana yaz. Bonus: en yüksek skoru bulan mantık ekle.

Tamamlama Kontrolü
  • Struct (OyuncuBilgi) tanımlandı
  • Map (anahtar→struct) oluşturuldu
  • Add ile oyuncular eklendi
  • Find ile anahtara göre erişildi
  • Struct pinleri Split/Break ile okundu
  • Bonus: en yüksek skor bulundu
Struct + Map kombinasyonuyla gerçek veri yapısı kurmak — profesyonel temel.
