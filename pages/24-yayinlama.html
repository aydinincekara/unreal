Yayınlama — Oyunu .exe Olarak Çıkarmak

Editör içinde çalışan bir oyun, yayınlanmış bir oyun değildir. Bu bölümde "Package Project" sürecini, build türlerini, platform ayarlarını, splash/icon, sürüm numarası ve dağıtım yöntemlerini öğreneceksin. Sonunda arkadaşına gönderebileceğin bir .exe'n olacak.

24.1

Packaging nedir? — Editörden dışarı

Geliştirme sırasında UE5 editörü tüm asset'leri, kodu ve motoru kendi içinde tutar. Oyun "Play" tuşuyla bu editörün içinde çalışır. Packaging, oyunu editörden ayırıp tek başına çalışabilecek bağımsız (standalone) bir uygulama'ya dönüştürme sürecidir.

Gerçek hayat benzetmesi

Bir aşçı tarif denerken mutfakta her şey var: bütün baharatlar, malzemeler, fırınlar. Bu "editör"dür. Ama yemeği müşteriye servis ederken sadece tabaktaki yemek gönderilir — mutfak değil. Packaging tam olarak bunu yapar: gereken tüm asset'leri, motor parçalarını ve kodu paketleyip "tabağı" oluşturur. Müşteri (oyuncu) UE5'i kurmadan oyunu oynar.

Paketleme sırasında neler olur?

Cooking
Asset'ler hedef platformun formatına dönüşür
Compile
C++ kod (varsa) ve Blueprint nativization
Stage
Çıkış dosyaları geçici klasöre toplanır
Pak
Asset'ler tek .pak dosyasına paketlenir
Package
Çalıştırılabilir .exe + bağımlılıklar oluşur

En sık unutulan ayar: Game Default Map

Bunu ayarlamazsan paketlenen oyun açıldığında boş veya yanlış level başlar. Genelde MainMenu level'ı buraya yazılır. "Oyunum çalışıyor ama bir tuşa basınca patlıyor" şikayetlerinin yarısı bu eksiklikten kaynaklanır.

Packaging — Build ayarları

Project Settings → Project → Packaging:

  • Build Configuration: DebugGame / Development / Shipping. Yayın için Shipping.
  • Build Target: Game (oyun) / Client / Server.
  • Use Pak File: AÇ — tüm asset'leri tek dosyada toplar (gizler ve hızlandırır).
  • Use Io Store: AÇ — yeni UE5 standardı, daha hızlı yükleme.
  • List of maps to include in a packaged build: dahil edilecek tüm level'lar listesi.
Map listesi neden gerekli?

UE5 paketlerken sadece başlangıç haritasından erişilebilen asset'leri otomatik dahil eder. Ana menüden Open Level ile geçilen başka level'lar varsa, bunları manuel olarak List of maps to include'a eklemelisin. Yoksa "level bulunamadı" hatası alırsın.

24.3

Icon ve Splash — Görsel kimlik

Oyunun .exe dosyasında görünen ikon ve açılışta gösterilen splash ekranı kişiselleştirilebilir. Windows platformu için iki ayrı yer:

Windows için

Project Settings → Platforms → Windows:

Game Icon
.ico dosyası — exe ikonu (256×256)
Splash Image
.bmp dosyası — açılış görseli (genelde 600×200)
.ico nasıl yapılır?

PNG'ni online bir dönüştürücüyle (örn. ICO Convert, icoconvert.com) .ico'ya çevir. 256×256, 128×128, 64×64, 32×32, 16×16 boyutlarını birden içeren bir multi-resolution .ico ideal — Windows hangi boyutu gösterirse kalitesi düşmez.

24.4

Build Configuration — Hangi mod?

Aynı projeden farklı amaçlar için farklı build'ler çıkar. Hata ayıklama testi yapan biriyle gerçek yayın yapan biri aynı yapıyı kullanmaz. UE5'te 4 standart konfigürasyon vardır:

ModHızBoyutDebugNe için?
DebugÇok yavaşBüyükTamMotor seviyesi C++ hata ayıklama
DebugGameYavaşBüyükKendi kodun içinGame kodu debug, motor optimize
DevelopmentHızlıOrtaSınırlıTest build, geliştirici inceleme, dahili dağıtım
Shipping ⭐En hızlıEn küçükYokYayın — kullanıcıya verilecek nihai build
"Test build" ile "Yayın build" karıştırma

Kullanıcıya her zaman Shipping build gönder. Development build'lerde geliştirici konsol, ekstra log, daha büyük boyut ve hafif güvenlik açığı vardır. Steam veya itch.io'ya yüklerken Shipping kullan.

İleri seviye — Test build

UE5'in Test diye gizli bir 5. konfigürasyonu daha var: Shipping kadar optimize, ama bazı debug imkanları açık. Sertifika öncesi (örn. konsol submit öncesi) son testler için kullanılır. UI'da görünmez, .Target.cs dosyasında ek ayar ister.

24.5

Paketleme adım adım — Windows .exe çıkışı

1
Build Configuration seç

Üst menü → Platforms → Windows → Build Configuration → Shipping. Bu seçim aktif kalır.

2
Build Target ayarla

Platforms → Windows → Build Target → [Senin Projen] (Game). Multiplayer için Client/Server ayrı.

3
Package Project

Platforms → Windows → Package Project. UE5 hedef klasör sorar — "Output" gibi bir klasör seç. Tıkla, paketleme başlar.

4
Bekle (Output Log)

Sağ altta Output Log mesajları akar. Tipik orta boy bir proje 5-30 dakika sürer. Büyük projelerde 1-2 saat. Kapatma! Hata olursa logda kırmızıyla görünür.

5
Tamamlandı → Klasörü aç

Sağ altta "Packaging complete!" mesajı. Output/Windows/ klasöründe oyunun .exe'si ve gerekli dosyalar. Çift tıkla — oyun bağımsız çalışır.

Çıkış klasörünün yapısı

Output/Windows/[ProjeAdı].exe — ana çalıştırılabilir.
Output/Windows/[ProjeAdı]/ — asset paketleri (.pak), motor dll'leri, plug-in'ler.
Dağıtım için hepsini birlikte gönder. Sadece .exe yetmez — yanındaki klasörler gerekli.

24.6

Sık hatalar — Paketleme bozulduğunda

İlk paketlemen büyük olasılıkla bir hata verecek. Korkma — herkesin başına gelir. Output Log'da kırmızı yazıları ara, en ilk hata mesajını oku (sonrakiler genelde domino etkisi).

En sık karşılaşılan 6 hata

"Missing Module" hatası

C++ modülü hatalı veya plug-in eksik. Sln dosyasını Visual Studio'da aç, Build → Rebuild. Hâlâ varsa Binaries ve Intermediate klasörlerini sil, projeyi yeniden generate et.

"Asset has missing reference"

Bir Blueprint'in referansları kırık. Window → Project → Asset Audit ile referansları kontrol et, eksikleri düzelt veya kaldır.

"Map not found"

Game Default Map ayarlanmamış veya level listesinde yok. Project Settings → Maps & Modes ve Packaging → maps to include kontrol et.

"Cooking failed for X"

Bir asset paketlemede başarısız. Genelde corrupt asset. Asset'i sahneden çıkarıp ayrıca aç, kaydet, yeniden dene. Çözmezse silmen gerekebilir.

"Disk space low"

Paketleme 10-50 GB ara dosya üretir. Diskte yer aç. Saved/StagedBuilds eski denemelerden dolu olabilir, sil.

".exe açılınca anında kapanıyor"

Crash. Saved/Logs/ içinde son log'a bak. Çoğunlukla eksik level, eksik C++ derleme, veya Game Default Map.

Sorunu izole etme

Karmaşık projede hata bulmak zorsa, küçük bir test paketi dene: yeni boş level → sadece bir küp → minimal GameMode. Bu paketleniyorsa sorun proje içeriğinde. Paketlenmiyorsa sorun motor/sistem seviyesinde.

24.7

Paket boyutunu küçültmek

UE5 projeleri büyüktür. Boş bir Third Person şablon bile paketlenirken 800 MB - 2 GB civarı çıkar. Bunu kullanıcıya indirme acısı yaşatmadan vermek için boyut optimizasyonu önemli.

Boyut küçültme yolları

  • Kullanılmayan asset'leri sil. Asset Audit → Unused → toplu sil. Sample projelerden gelen ekstralar büyük yer kaplar.
  • Compression: Project Settings → Packaging → Use Pak File ✓ + Create Compressed Cooked Packages ✓.
  • Texture boyutları gözden geçir. Gerçekten 4K texture lazım mı? 2K çoğu yere yeter.
  • Plug-in'leri kapat: Edit → Plug-ins → kullanmadıkların disable.
  • Audio: WAV yerine OGG/Vorbis kullan. Audio Compression import ayarı.
  • Maps to include filtrele. Test level'larını listeden çıkar.
  • Sample Content'i sil. Starter Content kalmışsa kaldır.
İleri seviye — Asset Cooking Statistics

Hangi asset paket boyutunu ne kadar şişirdiğini görmek için: pak'lama sonrası Saved/StagedBuilds/.../UAT/ klasöründeki CookerStats.csv dosyasını incele. Boyutu en büyük olanları bul → optimize et. Tipik suçlular: yüksek-çözünürlüklü texture, sıkıştırılmamış audio, uzun cinematic'ler.

24.8

Dağıtım — Oyunu insanlara ulaştır

.exe'n var, şimdi nasıl dağıtacaksın? Senaryoya göre farklı yollar:

Arkadaşlara test

Klasörü ZIP'le, Google Drive/WeTransfer/Discord'dan gönder. Hızlı, ücretsiz. Klasörün tamamını zip'le, sadece .exe yetmez.

itch.io

Indie sahnesinin merkezi. Ücretsiz yayın. Hesap aç, yeni game ekle, ZIP yükle. Sayfa kendine ait, tarayıcıdan oynanabilir HTML5 build da mümkün.

Steam

Profesyonel pazar. $100 Steam Direct ücreti. Steamworks SDK entegrasyonu gerekir. İnceleme + en az 30 gün hazırlık. En büyük kullanıcı kitlesi.

Epic Games Store

Daha az rekabet, daha küçük kitle. %88/12 paylaşım (Steam %70/30). UE5 ile uyumlu. Submission süreci Steam'den daha kapalı.

Telif lisansları kontrol et

Megascans Quixel = UE5 projelerinde ücretsiz, dış kullanım için lisans. Marketplace asset'leri her biri kendi lisansı ile gelir. Ses/müzik: royalty-free olduğundan emin ol veya kullanım hakkı al. Telif ihlali yayından kaldırma sebebidir.

Hap Bilgiler

  • Packaging = oyunu editörsüz çalışabilir hale getirme.
  • Game Default Map en sık unutulan ayardır — Project Settings → Maps & Modes.
  • Maps to include listesine ana menü dışındaki tüm level'ları ekle.
  • Build Configuration: Shipping — kullanıcıya verilecek build türü.
  • Pak File + Io Store açık olsun — sıkıştırma + hızlı yükleme.
  • Game Icon: .ico (multi-resolution), Splash: .bmp.
  • Dağıtımda sadece .exe değil, yanındaki tüm klasör gönderilir.
  • Hata olursa Output Log'a bak, ilk hata mesajından başla.
  • Asset Audit ile kullanılmayanları temizle — boyut küçülür.
  • Texture/audio sıkıştırması en büyük boyut tasarrufu sağlar.
  • itch.io başlangıç için ideal, Steam büyük pazar ($100 + 30 gün hazırlık).
  • Telif ve lisans kontrol: her asset/ses için kaynağı doğrula.

Pratik Projeler

İlk gerçek dağıtımını yap
Kolay~25 dk
Proje 1: Project Settings'i baştan sona doldur

Mevcut bir projende Project Settings → Project → Description'a tam kimlik bilgisi yaz: Display Title, Version (1.0.0), Company, Copyright. Maps & Modes'a doğru başlangıç level'ını ata. Packaging ayarlarını gözden geçir: Shipping seçili mi, Pak File açık mı, ilgili map'ler listede mi?

Tamamlama Kontrolü
  • Description sekmesi tam doldu
  • Game Default Map ayarlandı
  • Build Configuration: Shipping
  • Use Pak File ✓
  • Maps to include listesinde gerekli level'lar var
Paketleme öncesi profesyonel "checklist"'i alışkanlık hâline getirmek.
Orta~45 dk
Proje 2: İlk standalone .exe çıkışı

Basit bir test projesi paketle: Third Person şablonu + bir küp + ana menü level (sadece "Play" butonu). Project Settings tam doldur, ikon ekle (.ico üret), splash görsel ekle. Platforms → Windows → Package Project. Çıkan klasörden .exe'yi başka bir bilgisayara veya en azından başka kullanıcıya gönderip çalışıp çalışmadığını test et.

Tamamlama Kontrolü
  • Paketleme hatasız tamamlandı
  • .exe çift tıklayınca açıldı, oyun başladı
  • Splash görsel açılışta görünüyor
  • Pencerede doğru başlık ve ikon var
  • Ana menüden Play'e basınca oyun level'ı açılıyor
  • Hata logsuz kapanış (Alt+F4 veya quit ile düzgün çıkış)
End-to-end paketleme akışını başarıyla deneyimlemek.
İleri~75 dk
Proje 3: itch.io'ya gerçek yayın

Proje 2'nin .exe paketini zipleyip itch.io'ya yayınla: hesap aç, yeni game ekle, oyun adı/açıklama/cover image, ekran görüntüleri, fiyat (free de olabilir). ZIP'i yükle, "uploads" bölümünde Windows işaretle. Public butonuna basmadan önce "Draft" modunda kendin indirip kur ve sorunsuz çalıştığını doğrula. Bonus: README.txt ekle (sistem gereksinimleri, kontroller).

Tamamlama Kontrolü
  • itch.io hesabı aktif
  • Game sayfası açıldı (başlık, açıklama, görseller dolu)
  • ZIP yüklendi, Windows işaretli
  • Draft'tan kendin indirip kurabildin
  • Oyun .exe çalışıyor, hata yok
  • Public yayında veya "Restricted" testten geçti
  • Bonus: README ile sistem gereksinimleri ve kontroller belirtildi
İlk gerçek "yayınlanmış oyun" tecrübesi — geliştiriciden yayıncıya geçiş.
