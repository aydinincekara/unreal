Input & Hareket — Karakteri Sen Kontrol Et

Bir oyunda en çok hissedilen şey karakter hareketidir. Bu bölümde klavye/fare/gamepad girdilerini nasıl yakalayacağını, karakterini nasıl yürüteceğini, zıplatacağını ve kameranı nasıl döndüreceğini öğreneceğiz. Sonunda kontrol edebileceğin bir karakterin olacak.

7.1

Input nedir? — Oyuncunun sesini duymak

Input (girdi), oyuncudan oyuna gelen her sinyaldir: klavye tuşu, fare hareketi, gamepad tuşu, dokunmatik ekran dokunuşu. Oyun bu sinyalleri "yakalayıp" ne yapacağına karar verir. Input olmadan oyun olmaz — sadece bir film olur.

Gerçek hayat benzetmesi

Input sistemi kulak gibidir. Oyuncu konuşur (tuşa basar), oyun dinler ve cevap verir (karakter hareket eder). İyi bir kulak duyduğunu doğru anlar — "W" basıldı mı bıraktı mı, ne kadar süreyle basılı, gamepad analogu hangi yöne ne kadar itildi.

UE5'te iki sistem var: Eski vs Enhanced Input

Bu bölümde ne yapacağız?

Yeni başlayan için kafa karışmasın diye klasik tuş event'leri ile başlayacağız (W tuşuna basıldı, Space tuşuna basıldı...). Bu sistem hala çalışıyor ve mantığı net. Enhanced Input'un detayını Bölüm 22'de göreceğiz — orada modern, profesyonel yaklaşımı öğreneceksin.

7.2

İlk tuş event'i — Klasik yöntem

Bir Blueprint'in Event Graph'ında, sağ tıklayıp "Keyboard Events" kategorisine gidersen her tuşun event'ini görürsün. Bunlar 2 çıkışı olan kırmızı node'lardır:

Klavye event'i — anatomisi
Space Bar
Pressed (basıldı) ▸
Released (bırakıldı) ▸
// Pressed: tuş basıldığı an / Released: bırakıldığı an

Pressed pinine bağlarsan, oyuncu tuşa her bastığında bir kez çalışır. Released'a bağlarsan, tuşu bıraktığı anda çalışır. İkisi de aynı anda kullanılabilir.

Bu event nereye konmalı?

Klavye event'leri genelde Pawn / Character Blueprint'ine veya Player Controller'a konur. Sıradan bir Actor'da çalışmayabilir çünkü "input alma" ayarı kapalıdır. Character için hazırdır — Pawn değilse Details'te "Auto Receive Input"'u Player 0 yap.

7.3

Karakteri yürüt — İleri/geri/sağa/sola

Karakter Blueprint'i (Character class), zaten içinde CharacterMovementComponent taşır. Bu component yerçekimi, yürüme, zıplama hesaplarını otomatik yapar. Senin yapacağın tek şey "şu yöne git" demek. Bunu Add Movement Input node'u ile yaparsın.

1
Yeni Character BP oluştur

Content Browser → sağ tık → Blueprint Class → Character. İsim: BP_Karakter. Açtığında zaten içinde Capsule, Mesh, CharacterMovement hazır gelir.

2
İleri/geri eksen

Event Graph'ta W tuş event'i ekle. Pressed çıkışından Add Movement Input node'una bağla. World Direction'ın X'ini 1.0 yap. Aynı şeyi S tuşu için X=-1.0 yaparak ekle.

X = ileri/geri, Y = sağ/sol, Z = yukarı/aşağı.
3
Sağa/sola

D tuşu Pressed → Add Movement Input, World Direction Y = 1.0. A tuşu için Y = -1.0.

4
GameMode'a ata, sahneye koy

Bölüm 03'teki gibi GameMode'unun Default Pawn Class'ına BP_Karakter'i ata. Sahneye Player Start koy. Play'e bas — WASD ile karakterin yürümeli.

"Yürümüyor!" hatası

En sık sebepler: (1) Karakter sahnede yoksa GameMode → Default Pawn Class doğru atanmamış. (2) Tuş event'i yerine "Auto Receive Input" kapalı. (3) Add Movement Input yerine "AddActorWorldOffset" kullanılmış (yerçekimi bypass'lar, garip davranış).

İleri seviye — Aksis vs Buton

Klavye için tuş event'i yeterli ama gamepad analogu için "ne kadar itildi" bilgisini de almalısın (0.0 - 1.0 arası). Bunun için Enhanced Input'taki Axis Value kullanılır. Add Movement Input'un Scale Value pinine bu değeri verirsen yumuşak, analog hassasiyetli hareket olur. Detay Bölüm 22.

7.5

Kamera & Fare — Etrafına bakmak

Karakterin bakış yönünü kontrol etmek için fareyi kullanırız. Character BP'ye Spring Arm (kamera kolu) ve Camera component'leri eklenir. Sonra fare hareketiyle bu kol döndürülür.

1
Component'leri ekle

Components panelinde + Add → Spring Arm ekle. Sonra Spring Arm seçiliyken + Add → Camera ekle (Camera, Spring Arm'ın çocuğu olmalı). Spring Arm'ın Target Arm Length'i 400 yap (üçüncü şahıs mesafe).

2
Fare Yaw — sağa/sola dönme

Event Graph'ta "Mouse X" event'i ekle (Axis Events kategorisi). Çıkışındaki Axis Value'yu Add Controller Yaw Input node'una bağla.

3
Fare Pitch — yukarı/aşağı bakma

Mouse Y event'i → Add Controller Pitch Input. Y ekseni ters gelirse Axis Value'yu -1 ile çarp.

4
Spring Arm ayarı

Spring Arm seçili → Details → "Use Pawn Control Rotation" = true yap. Bu, kameranın Controller'ın dönüşünü takip etmesini sağlar.

Spring Arm neden var?

Spring Arm bir esnek olta gibidir. Ucuna kamera takılı. Karakterin arkasından uzanır. Karakter bir duvara sıkışırsa, Spring Arm otomatik kısalır — kamera duvarın içine girmez. Bu yüzden kamerayı doğrudan Character'a değil, Spring Arm'a takıyoruz.

Üçüncü şahıs vs Birinci şahıs

Üçüncü şahıs: Spring Arm uzun (400+), karakteri arkadan görürsün. Birinci şahıs: Spring Arm yok, Camera doğrudan karakterin başına (Mesh'in head socket'ine) bağlı. Mesh'i de "Owner No See" ayarıyla görünmez yaparsın (kendi vücudunu görmek istemiyorsan).

7.6

Hız ayarları ve koşma (Sprint)

Karakterin hızı CharacterMovement component'inin Max Walk Speed değeridir (varsayılan 600). Bu sayıyı oyun sırasında değiştirerek "koşma" mekaniği yaparsın:

Shift basılıyken koşma — basit yöntem
Left Shift
Pressed ▸
Released ▸
Set Max Walk Speed (1200)
hızlandır
// Released → 600 (normale döner)

Pressed: Get CharacterMovement → Set Max Walk Speed = 1200. Released: Set Max Walk Speed = 600. İki çizgili akış — biri hızlandırma, biri yavaşlatma.

İleri seviye — Stamina sistemi

Profesyonel sprint genelde stamina (nefes) ile sınırlanır. Float bir Stamina variable + Tick'te koşunca azalt, dururken artır. Stamina > 0 ise koşmaya izin ver, <= 0 olunca otomatik yavaşlat. UI'da bar göster (Bölüm 09). Bu, Bölüm 05'teki Custom Event ve Bölüm 06'daki Float variable bilgilerinin pratiğe döküldüğü yerdir.

Hap Bilgiler

  • Tuş event = Pressed + Released. Basıldığı an ve bırakıldığı an.
  • Add Movement Input ile yürüt — yerçekimi/collision otomatik.
  • X = ileri/geri, Y = sağ/sol, Z = yukarı/aşağı.
  • Character class CharacterMovementComponent ile gelir — hazır.
  • Jump ve StopJumping hazır node'lar — Space'e bağla yeter.
  • Max Jump Count = 2 → çift zıplama, kod yok.
  • Spring Arm kamerayı duvardan korur — kamerayı ona bağla.
  • Add Controller Yaw/Pitch fareyle dönüş için.
  • Max Walk Speed değiştirerek koş — varsayılan 600.
  • Karakter yürümüyor? GameMode → Default Pawn Class kontrolü, Auto Receive Input.

Pratik Projeler

Kendi kontrol edebildiğin karakteri yarat
Kolay~25 dk
Proje 1: WASD ile yürüyen karakter

Yeni BP_Karakter (Character) oluştur. W/A/S/D tuş event'leri + Add Movement Input ile 4 yöne hareket. GameMode'a ata, Player Start koy. Play'e bas, WASD ile etrafta yürü.

Tamamlama Kontrolü
  • Character BP oluşturuldu
  • 4 yön (W/A/S/D) event'leri eklendi
  • Add Movement Input doğru yönlerle bağlandı
  • GameMode'a Default Pawn olarak atandı
  • Player Start'tan WASD ile yürünebiliyor
Temel hareket sistemini elden kurmak.
Orta~40 dk
Proje 2: Çift zıplayan, fareyle bakan karakter

Proje 1'in üzerine: Spring Arm + Camera ekle (üçüncü şahıs). Mouse X/Y eksenleriyle bakış (Yaw/Pitch). Space ile Jump. CharacterMovement → Max Jump Count = 2 yap (çift zıplama). Test et.

Tamamlama Kontrolü
  • Spring Arm + Camera doğru hiyerarşide
  • Use Pawn Control Rotation aktif
  • Mouse X → Yaw, Mouse Y → Pitch çalışıyor
  • Space → Jump bağlı
  • Çift zıplama yapılabiliyor (Max Jump Count=2)
  • Kamera duvarda otomatik kısalıyor
Tam üçüncü şahıs kontrol şeması (hareket+kamera+zıplama).
İleri~60 dk
Proje 3: Sprint + Stamina sistemi

Proje 2 üstüne: Float Stamina variable (max 100). Shift basılıyken Max Walk Speed = 1200 + Tick'te Stamina'yı azalt (Delta × 30). Bırakınca Speed = 600 + Tick'te Stamina'yı artır (Delta × 20). Stamina ≤ 0 olunca otomatik koşmayı durdur (Branch). Ekrana Stamina'yı Print String ile yaz (UI Bölüm 09'da).

Tamamlama Kontrolü
  • Stamina Float variable (0-100) oluşturuldu
  • Shift Pressed/Released ile hız değişimi
  • Tick'te Stamina azalma/artma çalışıyor
  • Delta Seconds × oran kullanıldı (FPS bağımsız)
  • Stamina ≤ 0 → koşma otomatik kesiliyor
  • Print String ile değer görünüyor
Input + variable + Tick + Branch'ı gerçek bir mekanikte birleştirmek.
