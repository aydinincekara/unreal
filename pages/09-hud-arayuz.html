HUD & Arayüz — Ekran Üstü Bilgi

Oyuncunun canını, skorunu, mühimmatını nasıl görür? Ekranın üstüne çizilen bu bilgilere HUD denir. UE5'in UMG (Unreal Motion Graphics) sistemiyle can barı, skor göstergesi ve menü tasarlamayı, bunları canlı veriye bağlamayı öğreneceğiz.

9.1

HUD ve UMG nedir?

HUD (Heads-Up Display), oyun dünyasının üstüne çizilen 2D bilgi katmanıdır. Can barı, skor, mini harita, nişangah... Oyun 3D'de devam ederken bu bilgiler ekranda sabit durur. UE5'te HUD'ları UMG (Widget sistemi) ile tasarlarsın.

HUD nedir?

HUD, bir savaş uçağının cam göstergesi gibidir — pilot dışarı bakarken camın üstünde hız, irtifa, hedef bilgisi görür. Oyunda da oyuncu dünyayı görürken ekranın üstünde canını, skorunu görür. "Heads-Up" = "başını kaldırmadan" yani oyundan kopmadan bilgi.

Widget = arayüz parçası

UMG'de her arayüz öğesine Widget denir: buton bir widget, yazı bir widget, resim bir widget. Bunları bir Widget Blueprint içinde birleştirip ekrana çizersin. Tüm menüler, HUD'lar, pop-up'lar Widget Blueprint'tir.

9.2

İlk Widget'ını oluştur

Widget Blueprint editörü iki moda ayrılır: Designer (görsel tasarım) ve Graph (mantık).

UMG Designer — Yerleşim Şematik
1
Widget Blueprint oluştur

Content Browser → sağ tık → User Interface → Widget Blueprint. İsim: WBP_HUD. Çift tıkla — Designer açılır.

2
Öğeleri sürükle

Sol Palette'ten Progress Bar (can barı) ve Text (skor) öğelerini canvas'a sürükle. Konumlandır.

3
İsimlendir (önemli!)

Her widget'a Details'ten anlamlı isim ver: HealthBar, ScoreText. Ayrıca "Is Variable" kutusunu işaretle — yoksa Graph'tan erişemezsin.

9.3

Widget'ı ekrana çiz

Widget'ı tasarladın ama henüz ekranda görünmüyor. Onu ekrana eklemen gerek. En yaygın yer: karakterin veya Player Controller'ın BeginPlay'i.

BeginPlay → Create Widget node'u. Class = WBP_HUD. Bu, widget'ın bir kopyasını bellekte oluşturur (henüz görünmez).

3

Create Widget'ın Return'ünü bir variable'a (örn. HUDRef) Set et. Sonra HUD'u güncellemek istediğinde bu referansı kullanırsın.

9.4

Canlı veri bağlama — Can barı çalışsın

HUD görünüyor ama statik. Can azalınca barın da azalması lazım. Bunun iki yolu var: Binding (otomatik) ve manuel güncelleme (event ile).

Binding (Otomatik)

Progress Bar'ın Percent'ine bir fonksiyon bağlarsın. Her kare otomatik çağrılır, güncel canı döner. Kolay ama her kare çalışır (performans).

Kolay · sürekli

Manuel (Event)

Sadece can değiştiğinde HUD'a "barı güncelle" dersin. Daha verimli, profesyonel yöntem. Set Percent node'unu çağırırsın.

Verimli · olay bazlı
Can barı yüzdesi nasıl hesaplanır?

Progress Bar 0-1 arası değer alır (0 = boş, 1 = dolu). Yüzde = GüncelCan ÷ MaxCan. Örn. can 75, max 100 → 75÷100 = 0.75 → bar %75 dolu. Set Percent node'una bu float'ı verirsin.

İleri seviye — Binding mı, Event mı?

Binding kolaydır ama her kare çağrılır — 20 widget'ın hepsi her kare hesaplanırsa performans düşer. Profesyoneller Event Dispatcher kullanır: can değişince bir olay yayınlanır, HUD onu dinleyip sadece o an günceller. Çok daha verimli. Event Dispatcher'ları Bölüm 25'te detaylı göreceğiz. Başlangıçta binding'le başla, sonra event tabanlıya geç.

9.5

Skor metnini güncelle

Skor için Text widget'ını güncelleriz. Skor bir integer'dır, Text widget metin (Text) ister — arada dönüşüm gerekir.

1
Skor değişince

Skor arttığında (örn. altın toplama) HUD referansına eriş → ScoreText widget'ına ulaş.

2
Set Text + dönüşüm

ScoreText → Set Text (Text) node'u. Skor integer'ını ToText (Integer) ile çevir, sonuca "Skor: " metnini Append ile ekle.

9.6

Menü ve fare imleci

Bir menü (örn. duraklatma) açtığında fare imlecinin görünmesi ve butonlara tıklanabilmesi gerekir. Bunun için Input Mode ayarlanır.

Input ModeNe yapar?Ne zaman?
Game OnlySadece oyun kontrolü, imleç gizliNormal oynanış
UI OnlySadece arayüz, imleç açıkAna menü, tam ekran menü
Game and UIİkisi birden, imleç açıkEnvanter, kısmi menü
İmleç görünmüyor / butona tıklanamıyor?

Menü açtın ama fare yok veya butonlar tepki vermiyorsa: Set Input Mode UI Only (veya Game and UI) çağır ve Set Show Mouse Cursor = true yap. Menüyü kapatırken tersine Set Input Mode Game Only + imleci gizle.

Hap Bilgiler

  • HUD = ekran üstü 2D bilgi (can, skor). UMG ile yapılır.
  • Widget Blueprint: Designer (görsel) + Graph (mantık) iki modlu.
  • "Is Variable" işaretle — yoksa widget'a Graph'tan erişemezsin.
  • Create Widget → Add to Viewport ile ekrana çizilir.
  • Widget referansını sakla (variable) — sonra güncellemek için.
  • Progress Bar 0-1 arası; yüzde = GüncelCan ÷ MaxCan.
  • Integer → ToText ile skor metne çevrilir.
  • Binding = otomatik (her kare), Event = verimli (sadece değişince).
  • Menüde imleç için Set Input Mode UI/Game and UI + Show Mouse Cursor.

Pratik Projeler

Oyununa arayüz kat
Kolay~25 dk
Proje 1: Skor göstergesi

WBP_HUD oluştur, bir Text widget koy (ScoreText). BeginPlay'de Create Widget + Add to Viewport. Altın topladıkça skoru artır ve Set Text ile ekrandaki yazıyı güncelle.

Tamamlama Kontrolü
  • Widget Blueprint oluşturuldu
  • Text widget "Is Variable" işaretli
  • Ekrana eklendi (Add to Viewport)
  • Skor toplandıkça güncelleniyor
  • Integer→Text dönüşümü yapıldı
Widget oluşturma, ekrana ekleme, metin güncelleme.
Orta~40 dk
Proje 2: Çalışan can barı

HUD'a bir Progress Bar (HealthBar) ekle. Karakterde Health (float, max 100) variable'ı olsun. Bir hasar tuşuyla (örn. H) canı 10 azalt. Can değişince barın Percent'ini GüncelCan÷MaxCan ile güncelle. Bar canla birlikte azalsın.

Tamamlama Kontrolü
  • Progress Bar eklendi (HealthBar)
  • Health variable var (max 100)
  • Hasar alınca can azalıyor
  • Bar yüzdesi doğru hesaplanıyor (0-1)
  • Bar görsel olarak azalıyor
Progress Bar'ı canlı veriye bağlamak.
İleri~60 dk
Proje 3: Duraklatma menüsü

Bir WBP_PauseMenu yap (Devam Et + Çıkış butonları). Esc tuşuna basınca: menüyü Create+AddToViewport, Set Game Paused = true, Input Mode UI Only, imleç göster. "Devam Et" butonu menüyü kapatıp oyunu sürdürsün (Remove from Parent, unpause, Game Only). Butonlara OnClicked event'i bağla.

Tamamlama Kontrolü
  • Pause menü widget'ı 2 butonlu
  • Esc ile açılıyor, oyun duruyor (Set Game Paused)
  • Input Mode + imleç doğru ayarlandı
  • Butonların OnClicked'i çalışıyor
  • Devam Et menüyü kapatıp oyunu sürdürüyor
  • Çıkış oyunu kapatıyor (Quit Game)
Tam işlevsel menü: pause, input mode, buton event'leri.
