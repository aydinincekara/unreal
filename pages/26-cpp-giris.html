C++ Giriş — Motorun Dilini Öğren

Blueprint UE5'in görsel yüzü; C++ ise motorun kendi dili. Bu bölüm sıfırdan başlar: algoritma nedir, syntax nedir, neden tipler vardır. Sonra Visual Studio kurulumu, UE5'in özel makroları (UCLASS, UFUNCTION, UPROPERTY) ve C++↔Blueprint köprüsü. Sonunda kendi C++ class'ını yazıp Blueprint'ten kullanıyor olacaksın.

26.1

Neden C++? — Blueprint yetmediğinde

Blueprint güçlüdür ve UE5 oyunlarının büyük çoğunluğu sadece Blueprint ile yazılabilir. Ama belli noktalarda C++ devreye girer:

Performans
C++ Blueprint'ten ~10x hızlı çalışır
Karmaşık sistem
Save sistemi, network, kompleks AI
Motor seviyesi
Yeni asset türü, custom render
Üçüncü parti kütüphane
REST API, database, fizik kütüphanesi
Ekip çalışması
Git ile merge — C++ text, BP binary
AAA üretim
Çoğu studyo C++ ve BP'yi birlikte kullanır
Gerçek hayat benzetmesi

Blueprint = otomatik vites otomobil. Bir mühendis günde 100 km yol gidebilir, basit ve hızlı. C++ = F1 yarış arabası — daha hızlı, daha güçlü, ama dişlileri/turbo'yu/diferansiyeli kendin ayarlarsın. Büyük projeler için çoğu zaman ikisi birlikte kullanılır: çekirdek (engine room) C++'tır, dış katman (gameplay tweak) Blueprint'tir.

C++ ile Blueprint birbirinin rakibi değil

"C++ mi BP mi?" yanlış soru. Doğrusu: "C++ ne için, BP ne için?". Genelde: temel sınıflar (Character, Weapon, AI Controller) C++'ta yazılır, sonra bunların varyantları (BP_AKMain, BP_Pistol) Blueprint'te türetilir. Programcı C++ tarafında çalışır, tasarımcı BP tarafında değer ayarlar.

26.2

Algoritma nedir? — Sıfırdan giriş

Programlama bilmiyorsan bu kısım önemli. Algoritma, bir sorunu çözmek için sıralı adımlar dizisidir. Programlama dili (C++ vs.) ise bu algoritmayı bilgisayara anlatma yolundan biri.

Gerçek hayat benzetmesi

Algoritma = yemek tarifi. "1. Tencereye 2 litre su koy. 2. Suyu kaynat. 3. Makarnayı at. 4. 8 dakika bekle. 5. Süz." — bu bir algoritmadır. Programlama dili ise tarifin yazıldığı dildir (Türkçe / İngilizce / Fransızca). Tarif değişmez, dili değişebilir. Bilgisayar Türkçe tarif okuyamaz, ama C++ konuşur.

Bir programın 3 temel yapı taşı

1. Variable (Değişken)

Bir değeri saklayan kutu. int can = 100; "can adında bir kutu var, içinde 100 yazıyor". Hareket, hız, isim, skor — hepsi değişken.

2. Conditional (Koşul)

"Eğer X ise A yap, değilse B yap". if (can <= 0) { öl(); }. Karar mekanizması.

3. Loop (Döngü)

"Şunu N defa tekrar et". for (int i=0; i<10; i++) { vur(); }. 10 defa vur. Verileri tek tek işleme.

Function (Fonksiyon)

Bu 3 yapıyı bir araya getirip "bir işi yapan paket" oluşturursun — buna function denir. Yemek tarifindeki "Soğanı doğra" tek başına bir mini-tariftir, ana yemek tarifine çağrılır.

Function = Blueprint'in zaten bildiğin "fonksiyon"u
void HasarAl(int miktar)        // Function: HasarAl, parametre: miktar
{
    can = can - miktar;          // can azalt
    if (can <= 0)                 // koşul
    {
        Oldu();                  // başka function çağır
    }
}

Bunu Blueprint'le karşılaştır: bir HasarAl function node'u, input pin'i (Miktar), içinde "Set Can" + "Branch" + "Oldu" node'ları... Aynı şey, yazılı ifade. C++ "yazılı Blueprint" gibi düşünülebilir.

26.3

C++ syntax temelleri — Hızlı tur

Bu, derin bir C++ kursu değil. UE5'te okuyabilmek ve değiştirebilmek için yeterli minimum bilgi.

Variable türleri (Blueprint'tekinin aynısı)

Temel veri türleri
int    sayi   = 5;          // Tam sayı
float  hiz    = 12.5f;      // Ondalıklı (Blueprint'te 'Float')
bool   diri   = true;       // true/false
FString isim = TEXT("Ali"); // UE5'te özel string türü
FVector konum = FVector(0,0,100); // 3D vektör (X,Y,Z)

Dikkat: Standart C++'ta std::string kullanılır, ama UE5'in kendi string türü FString'tir. UE5'te neredeyse her tür F (struct), U (UObject) veya A (Actor) öneki ile başlar — bu UE5'in kendi konvansiyonudur.

Operatörler

OperatörAnlamıÖrnek
=Atama (kutuya değer koy)can = 100;
==Eşit mi? (karşılaştırma)if (can == 0)
!=Eşit değil mi?if (isim != "Ali")
<, >, <=, >=Karşılaştırmaif (can <= 0)
+, -, *, /Aritmetikcan = can - 10;
+=Üstüne ekle (kısa yol)skor += 5;
&&VE (mantıksal)if (canli && silahli)
||VEYA (mantıksal)if (oldu || teslim)
!DEĞİL (tersi)if (!gizli)
En sık başlangıç hatası: = vs ==

= atamadır (değer koy). == karşılaştırmadır (eşit mi sor). if (can = 0) yazarsan can'i 0'a atayıp koşulu hep "yanlış" yapar, kafa karıştırıcı bug çıkarır. if içinde her zaman == kullan.

Anatomi — basit bir fonksiyon

C++ fonksiyon anatomisi
//    [return tipi] [isim]([parametreler])
       int          Topla   (int a, int b)
       {                                        // ← fonksiyon başlangıcı
           int sonuc = a + b;                   // hesapla
           return sonuc;                        // dönüş değeri
       }                                        // ← fonksiyon bitişi

// Kullanım:
int x = Topla(3, 5);                            // x artık 8

void = "hiçbir şey döndürmüyor" demek (Blueprint'te ucu pin'siz function). return olan fonksiyonlar değer üretir, output pin'li function gibi.

26.4

Visual Studio kurulumu — UE5 için yapılandırma

UE5'te C++ yazmak için Visual Studio Community 2022 (ücretsiz, Windows) veya Rider for UE (ücretli, çapraz platform) gerekir. Çoğu yeni başlayan VS Community kullanır.

1
Visual Studio Community 2022 indir

visualstudio.microsoft.com/vs/community/ adresinden indir. Free, kişisel kullanım için açık.

2
Workload seçimi — KRİTİK

Yüklerken Workloads ekranı çıkar. Şunları işaretle:

  • "Game development with C++" ← en önemli
  • "Desktop development with C++"
  • "Game Development with C++" altında: "Unreal Engine installer" ve "Diagnostic Tools"
3
Individual Components

"Individual components" sekmesine geç, ara: "Windows 10/11 SDK" (en yenisi), "MSVC v143 - x64/x86 build tools" işaretli olmalı. Bu UE5 derlemesi için zorunlu.

4
Install + bekle

~15-30 GB indirme. Bittikten sonra bilgisayarı yeniden başlat.

5
UE5'i C++ projesine ayarla

Yeni proje açarken sol üst köşede "Blueprint" ve "C++" seçenekleri vardır. C++'ı seç. Mevcut Blueprint projesini de C++'a dönüştürebilirsin: Tools → New C++ Class.

"Compiler not found" hatası alırsan

UE5 derleyemiyorsa Workload eksiktir. Visual Studio Installer'ı tekrar aç → Modify → Workloads'ı kontrol et. "Game development with C++" ve "Unreal Engine installer" işaretli olmalı.

26.5

İlk C++ class'ın — Adım adım

Çok basit bir C++ Actor sınıfı oluşturup, içine bir değişken ve bir function koyacağız. Sonra Blueprint'ten kullanacağız.

1
Tools → New C++ Class

UE5 editöründe üst menü → Tools → New C++ Class. Parent Class olarak Actor seç. Next.

2
İsim ver

Name: MyCustomActor. Public/Private: Public (default). Create Class'a bas. UE5 dosyaları üretir ve Visual Studio'yu açar.

3
İki dosyayı incele

Solution Explorer'da iki dosya açılır:

  • MyCustomActor.h — header (deklarasyon: ne var?)
  • MyCustomActor.cpp — implementation (nasıl çalışıyor?)

Header (.h) — değişken ve function ekle

MyCustomActor.h (ilgili kısım)
UCLASS()
class MYPROJECT_API AMyCustomActor : public AActor
{
    GENERATED_BODY()

public:
    AMyCustomActor();                              // Constructor

    // ★ Blueprint'ten görünür ve değiştirilebilir değişken
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Stats")
    float Can = 100.0f;

    // ★ Blueprint'ten çağrılabilir fonksiyon
    UFUNCTION(BlueprintCallable, Category = "Combat")
    void HasarAl(float Miktar);

protected:
    virtual void BeginPlay() override;
};

Implementation (.cpp) — function'ı yaz

MyCustomActor.cpp
#include "MyCustomActor.h"

AMyCustomActor::AMyCustomActor()
{
    PrimaryActorTick.bCanEverTick = true;
}

void AMyCustomActor::BeginPlay()
{
    Super::BeginPlay();
    UE_LOG(LogTemp, Warning, TEXT("Aktör doğdu! Can: %f"), Can);
}

void AMyCustomActor::HasarAl(float Miktar)
{
    Can -= Miktar;
    UE_LOG(LogTemp, Warning, TEXT("Hasar aldı! Kalan can: %f"), Can);

    if (Can <= 0.0f)
    {
        Destroy();   // Aktörü sahneden sil
    }
}

Derle (Compile)

1
Visual Studio'da Build

VS'de Build → Build Solution (Ctrl+Shift+B). Veya UE5 editörüne dönüp altta Compile butonu (sarı yıldız).

2
UE5'te kullan

Compile başarılıysa Content Browser → C++ Classes klasörünü aç. MyCustomActor sınıfını sahneye sürükle veya ondan bir Blueprint türet: sağ tık → Create Blueprint class based on MyCustomActor.

26.6

UE5 makroları — Sihirli kelimeler

C++ kodunda gördüğün UCLASS(), UFUNCTION(), UPROPERTY() — bunlar UE5'in özel makroları. Editor'a "bu kodu Blueprint'le de paylaş, garbage collection'a dahil et, network'te replicate et" gibi şeyleri söylerler. UE5'in C++ ile entegrasyonu bu makrolarla yapılır.

3 ana makro

UCLASS()

"Bu class UE5'in sınıf sistemine kayıtlı olsun" — Blueprint'ten görünür, refleksiyon (otomatik tanıma) yapılabilir, garbage collected olur.

Class üstüne

UFUNCTION()

"Bu function Blueprint'e açık olsun, ya da replicate edilsin, ya da Server only çalışsın" — Blueprint nodeu olarak çıkar.

Function üstüne

UPROPERTY()

"Bu variable editör panelinde görünsün, BP'den okunsun, kaydedilsin" — Details panelinde editlenebilir alan olur.

Variable üstüne

UPROPERTY parametreleri — En sık olanlar

ParametreNe yapar?
EditAnywhereDetails panelinde her örnekte editlenebilir
EditDefaultsOnlySadece Blueprint default değerleri için
EditInstanceOnlySadece sahnedeki örnek için (CDO değil)
VisibleAnywhereGörünür ama düzenlenemez (debug için)
BlueprintReadOnlyBP'den okuyabilir ama yazamaz
BlueprintReadWriteBP'den hem oku hem yaz
Category = "X"Details panelinde hangi grupta görünsün
ReplicatedMultiplayer'da network'te paylaşılsın

UFUNCTION parametreleri

ParametreNe yapar?
BlueprintCallableBlueprint'ten çağrılabilir node
BlueprintPureYan etkisiz, exec pin'siz (Pure function)

İleri seviye — Reflection sistemi

UE5'in makro sistemi aslında bir reflection (yansıma) motorudur. Bu makrolar derleme öncesi Unreal Header Tool tarafından okunur ve ekstra C++ kodu üretilir (örn. .generated.h dosyaları). Bu sayede C++ class'ları runtime'da kendilerini "tanıyabilir" — Blueprint'in C++ ile konuşabilmesinin altyapısı budur.

26.7

C++ ↔ Blueprint köprüsü — Hibrit mimari

C++'ı tek başına kullanmazsın. Hibrit mimari en yaygın yaklaşımdır: C++ taban sınıfı yazar, Blueprint ondan türetir ve ayarlar.

Hibrit mimari katmanları C++ taban + BP varyantları

Pratikte ne olur?

  • C++'ta: Mantığı yaz — Fire(), Reload(), TakeDamage(), Stats hesapları. Performans kritik kod.
  • Blueprint'te: C++ class'tan türet (Create BP class based on ABaseWeapon). UPROPERTY ile açtığın değişkenleri ayarla: Mesh, Damage, Sound, VFX, Animation.
  • Tasarım iterasyonu Blueprint'te yapılır (hızlı). Sistem değişiklikleri C++'ta (sağlam).
BlueprintImplementableEvent — En zarif köprü

C++'ta function tanımlarsın ama içini doldurmazsın — UFUNCTION(BlueprintImplementableEvent). Blueprint o function'ı override eder, kendi node'larıyla doldurur. Tasarımcı VFX/Animation tetiklemesini Blueprint'te yapar; programcı sadece "ne zaman çağrılacak"ı C++'tan kontrol eder. AAA studyoların en sık kullandığı patterndir.

Hap Bilgiler

  • C++ ile Blueprint rakip değil, birlikte çalışan tamamlayıcı katmanlardır.
  • Algoritma = sıralı adımlar, programlama dili sadece o adımları bilgisayara anlatma yolu.
  • 3 yapı taşı: Variable, Conditional, Loop. Function bunları paketler.
  • = atama, == karşılaştırma. Karıştırma — bug çıkar.
  • UE5'in özel string'i FString, standart C++ string değil.
  • U/A/F öneki UE5 konvansiyonu (UObject, Actor, struct).
  • Visual Studio Community + "Game development with C++" workload zorunlu.
  • .h dosyası = ne var (deklarasyon), .cpp = nasıl çalışıyor (implementation).
  • UCLASS, UFUNCTION, UPROPERTY UE5'in sihirli makroları — Blueprint köprüsü.
  • UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="X") en sık kombinasyon.
  • UFUNCTION(BlueprintCallable) Blueprint'te çağrılır.
  • Hibrit mimari: C++ taban sınıf + BP varyantları — endüstri standardı.
  • BlueprintImplementableEvent = C++ ilan eder, BP implementsr.

Pratik Projeler

İlk C++ kodunu yaz
Kolay~35 dk
Proje 1: Visual Studio kurulumu + ilk C++ class

Visual Studio Community 2022'yi "Game development with C++" workload'u ile kur. Yeni bir UE5 C++ projesi aç (veya mevcut BP projeyi C++'a çevir: Tools → New C++ Class). Parent: Actor. İsim: MyCustomActor. Otomatik üretilen .h ve .cpp'yi VS'de aç, incele. UE5 editöründen Compile butonuna bas, başarılı sonuç al.

Tamamlama Kontrolü
  • VS 2022 + Game development workload kuruldu
  • UE5'te C++ Class oluşturuldu
  • VS açılıp .h ve .cpp dosyaları görüldü
  • Compile başarılı (hata yok)
  • C++ Classes klasöründe sınıf görünüyor
C++ geliştirme ortamını kurmak ve ilk derleme deneyimini yaşamak.
Orta~50 dk
Proje 2: UPROPERTY + UFUNCTION ile Can sistemi

MyCustomActor'a şunları ekle: UPROPERTY ile float Can = 100.0f; ve float MaksimumCan = 100.0f;. UFUNCTION(BlueprintCallable) ile void HasarAl(float Miktar) — can azaltır, 0 olursa Destroy() çağırır. UFUNCTION(BlueprintPure) ile float CanOrani() — Can/MaksimumCan döner. Compile, sahneye sürükle, C++'tan türetilmiş bir BP yarat, BP'de değerleri ayarla. Başka bir BP'den (örn. trigger) HasarAl(25) çağır.

Tamamlama Kontrolü
  • 2 UPROPERTY (Can, MaksimumCan) eklenmiş — Details panelinde görünüyor
  • HasarAl UFUNCTION'u Blueprint'ten çağrılabilir node olarak görünüyor
  • CanOrani BlueprintPure olarak görünüyor (yeşil node)
  • Compile hatasız
  • BP'den HasarAl çağrıldığında Can azalıyor
  • Can 0'a düşünce aktör Destroy oluyor
UE5'in C++ ↔ Blueprint makrolarını ilk elden uygulamak.
İleri~90 dk
Proje 3: Hibrit silah mimarisi

C++'ta ABaseWeapon : public AActor taban sınıfı yaz. İçinde: UPROPERTY'ler — Damage, Range, FireRate, MagazineSize, MeshComponent. UFUNCTION'lar — Fire() (line trace yap, hit'e damage uygula), Reload(). Ek olarak UFUNCTION(BlueprintImplementableEvent) ile OnFireEffects() — sadece deklarasyon, BP doldurur. Sonra Blueprint'te 3 türevini yarat: BP_Pistol, BP_Shotgun, BP_Rifle — her birinin Damage/Range/Fire Rate/Mesh ayarları farklı, OnFireEffects BP override'ında her biri kendi muzzle flash + sound çağırıyor.

Tamamlama Kontrolü
  • ABaseWeapon C++ class yazıldı, derleme başarılı
  • Damage, Range, FireRate, MagazineSize UPROPERTY olarak BP'den görünüyor
  • Fire() line trace yapıyor, damage uyguluyor
  • OnFireEffects BlueprintImplementableEvent olarak doğru şekilde tanımlı
  • 3 BP türev oluşturuldu (Pistol/Shotgun/Rifle)
  • Her birinin değerleri farklı
  • Her birinde OnFireEffects override edildi (farklı VFX/SFX)
  • 3 silah sahnede test edildi, hepsi farklı davranıyor
Endüstri standardı hibrit mimari deneyimi — bir AAA projenin nasıl yapılandırıldığını anlamak.
