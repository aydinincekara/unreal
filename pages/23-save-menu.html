Save & Menü — Oyun Verilerini Saklamak ve Menüler

Oyununda ilerlemeyi kaydetmek, ana menü kurmak, pause menüsü göstermek, seviyeler arası geçmek... Bu bölümde SaveGame asset ile veri kayıt/yükleme, Main Menu Level mimarisi, Pause sistemi ve Level streaming/transition akışını öğreneceksin.

23.1

SaveGame asset — Kayıt nesnesi

UE5'te kayıt sistemi SaveGame adlı özel bir Blueprint class'a dayanır. Bu sınıfı miras alırsın, içine kaydetmek istediğin değişkenleri yazarsın. Sonra bu nesne diske .sav dosyası olarak yazılır/okunur.

Gerçek hayat benzetmesi

SaveGame = bir kasa. İçine ne koymak istersen onu koyarsın (altın miktarı, sağlık, mevcut level adı, oyuncu konumu). Sonra kasayı kilitler diske kaldırırsın. Tekrar açtığında kasanın içeriği aynen yerinde. Sadece sen ne koyduysan o saklanır — bütün dünyayı kaydetmez, sen ne istersen onu kaydeder.

SaveGame Blueprint oluşturma

1
Yeni Blueprint Class

Content Browser'da sağ tık → Blueprint Class. Parent Class olarak SaveGame ara, seç. İsim: BP_GameSave.

2
Variable'lar tanımla

Açtığında "Save_Slot_Name" gibi varsayılan değişkenler görürsün. Ek olarak istediğin variable'ları ekle: PlayerHealth (Float), PlayerLocation (Vector), Gold (Integer), CurrentLevel (Name), Inventory (Array of Struct).

3
Compile + Save

Kaydet. Bu artık kullanılmaya hazır.

Ne kaydetmeli, ne kaydetmemeli?

Kaydet: oyuncu istatistikleri, envanter, ilerleme bayrakları, ayarlar, level adı, konumu, sayaçlar.
Kaydetme: sahnedeki Actor referansları (yeniden yüklenmez), büyük asset'ler, runtime objeleri. Bunlar tekrar yüklendiğinde geçersiz olur.

İleri seviye — Async Save/Load

Büyük save dosyaları için Async Save Game to Slot ve Async Load Game from Slot kullan. Bunlar disk I/O'sunu arka planda yapar, ana thread'i bloklamaz. Console oyunlarında genelde zorunludur (PS5/Xbox sertifikasyon kuralları senkron save'i reddedebilir).

Slot ismi stratejisi

"Slot1", "Slot2", "Slot3" ile birden çok kayıt slotu yapabilirsin. Veya isim üret: "AutoSave_" + Tarih. Settings ayrı bir slot olur ("Settings"). User Index 0 standart oyuncudur; split-screen'de farklı oyuncular için farklı index kullanılır.

23.4

Ana Menü — Ayrı bir Level

Profesyonel oyunlarda Ana Menü ayrı bir Level'dır. Bunu yapmanın iki nedeni var: (1) menü görseli oyun sahnesinden farklı/sade olur, (2) menü açıkken oyun dünyası yüklü değildir — RAM tasarrufu.

Mimari

L_MainMenu
Boş veya basit sahne — sadece bir kamera ve UI
WBP_MainMenu
Widget — butonlar (Yeni Oyun, Devam, Ayarlar, Çıkış)
GM_MenuGameMode
Sadece menü için özel GameMode (default oyuncu input'u devre dışı)
L_Game
Asıl oyun level'ı (veya birden fazla)

L_MainMenu kurulumu

1
Yeni Level oluştur

File → New Level → Empty Level. Kaydet: L_MainMenu.

2
Background hazırla

İstersen statik bir kamera + dönen 3D karakter, ışık, fog. Veya tek renk arka plan.

3
Level Blueprint'te Widget göster

Level Blueprint → BeginPlay → Create Widget (WBP_MainMenu) → Add to Viewport. Sonra Set Show Mouse Cursor (true) + Set Input Mode UI Only.

4
World Settings → GameMode

Sahne için özel GM_MenuGameMode atayarak ana oyun GameMode'u burada çalışmasın.

23.5

Level geçişleri — Open Level + Transition

Bir level'dan diğerine geçiş, oyunun en sık yapılan işlerinden biri. Üç yöntem var:

Open Level (klasik)

Mevcut level kapatılır, yenisi yüklenir. Yükleme ekranı genelde gerekir. Basit ve hızlı uygulama.

Level Streaming

Birden fazla sublevel aynı dünyada tutulur, ihtiyaca göre yüklenir/boşaltılır. Açık dünya oyunları için.

World Partition

UE5'in açık dünya çözümü — dünya otomatik gridlere bölünür, dinamik yüklenir. Manuel sublevel yönetimi yok.

Yükleme ekranı — Persistent Widget

Open Level çağrılır çağrılmaz mevcut level dump edilir, yenisi yüklenirken kara ekran kalır. Game Instance üzerinden bir yükleme widget'ı göster ve "Loading..." göster. Game Instance level değişikliklerinde silinmediği için ideal.

23.6

Pause sistemi — Oyunu durdur

Pause menüsü = oyun dünyasını dondur + UI göster. UE5'te Set Game Paused (true) tek başına yeterlidir; tüm Tick'leri ve fiziği durdurur.

Pause akışı

1
IA_Pause + Esc

IMC_OnFoot'a Esc tuşu → IA_Pause ekle (Trigger: Pressed).

2
Blueprint: Pause Toggle

Karakter veya Controller Blueprint'te IA_Pause Triggered → Is Game Paused? → Branch → True ise Set Game Paused (false) + RemoveFromParent (WBP_Pause), False ise Set Game Paused (true) + Create Widget WBP_Pause + Add To Viewport.

3
Mouse + Input Mode

Pause AÇIK iken: Set Show Mouse Cursor true + Set Input Mode Game and UI. Pause KAPALI iken tersi.

Pause oyun dünyasını durdurur ama bir Actor'ün hâlâ çalışmasını istiyorsan (örn. menü animasyonu, müzik) Actor Details'te Tick When Paused tikini aç veya Widget'da Behavior → Force Tick While Paused ile bunu sağla.

23.7

Game Instance — Level'lar arası kalıcı veri

Open Level çağrıldığında her şey yeniden başlatılır — karakter, GameMode, level içindeki Actor'ler... Hepsi yok edilir, yenisi oluşturulur. Ama bazı veriler level boyunca kalıcı olmalı (mevcut altın, oyuncu adı, mevcut quest). İşte burada Game Instance devreye girer.

1
BP_GameInstance oluştur

Blueprint Class → Parent: GameInstance. İsim: BP_GameInstance.

2

CurrentSaveGame (BP_GameSave ref), PlayerProfile (struct), VolumeMaster (float).

Project Settings → Maps & Modes → Game Instance Class = BP_GameInstance. Bundan sonra oyun bu instance ile başlar.

4
Her yerden eriş

Blueprint'te Get Game Instance → Cast to BP_GameInstance. Sonra Get/Set ile değişkenlere ulaş.

İleri seviye — Subsystem'lar

GameInstance'a sürekli yeni şey eklemek karmaşıklaştırır. UE5'in Game Instance Subsystem sistemi modüler alternatif sunar: her özellik için ayrı bir Subsystem class (C++ veya Blueprint) — ödeme, achievement, ses, ağ. Otomatik instance üretilir, GameInstance'tan erişilir.

Hap Bilgiler

  • SaveGame class miras al, içine kaydetmek istediğin variable'ları koy.
  • Create Save Game Object → Set'le → Save Game to Slot kayıt sırası.
  • Load Game from Slot → Cast to BP_GameSave yükleme sırası.
  • Does Save Game Exist ile kayıt var mı kontrol et.
  • Slot Name + User Index ile birden çok kayıt slotu.
  • Async Save/Load büyük dosyalar için tercih.
  • Sahnedeki Actor referansı kaydedilmez, sadece veri.
  • Ana Menü ayrı bir Level'dır: L_MainMenu + WBP_MainMenu.
  • Project Settings → Game Default Map = L_MainMenu (packaged başlangıç).
  • Open Level klasik geçiş; Level Streaming ve World Partition açık dünya için.
  • Set Game Paused tek satırla oyunu dondurur.
  • Tick When Paused ile pause'da bile çalışan Actor.
  • Game Instance level'lar arası kalıcı veri kutusu.
  • Subsystem'lar modüler GameInstance alternatifi.

Pratik Projeler

Tam kaydet/yükle + menü altyapısı kur
Kolay~30 dk
Proje 1: Health + Gold kayıt sistemi

BP_GameSave oluştur: PlayerHealth (float), Gold (int) variable'ları. Karakter Blueprint'inde F5 tuşuna basıldığında SaveProgress çağrılsın (mevcut Health + Gold kaydedilsin). F9'a basılınca LoadProgress çağrılsın (kaydedilen veri karaktere yüklensin). Sahnede Health ve Gold'u HUD'da göster, basıp doğruluğu test et.

Tamamlama Kontrolü
  • BP_GameSave oluşturuldu
  • F5 kaydet, F9 yükle çalışıyor
  • Health ve Gold doğru saklanıyor
  • Cast to BP_GameSave kullanılıyor
  • Does Save Game Exist kontrolü var
Save/Load çekirdek akışını ezbere geçirmek.
Orta~55 dk
Proje 2: L_MainMenu + Pause sistemi

L_MainMenu boş bir level olarak oluştur, içine kamera + arkaplan. WBP_MainMenu butonları: Yeni Oyun, Devam, Çıkış. Yeni Oyun → L_Game'i aç. Devam → SaveGame var mı kontrol et + aç. Ayrıca pause sistemi: oyun içinde Esc → WBP_Pause aç, Set Game Paused true. Pause widget'tan Ana Menüye Dön + Devam Et + Çıkış.

Tamamlama Kontrolü
  • L_MainMenu kuruldu, WBP_MainMenu görünüyor
  • Yeni Oyun L_Game'i açıyor
  • Devam butonu Does Save Game Exist kontrol ediyor
  • Esc ile pause açılıp kapanıyor
  • Pause'dan Ana Menüye Dön çalışıyor
  • Mouse cursor + Input Mode doğru ayarlı
Tam bir oyun başlangıç akışı: menüden oyuna, oyundan pause'a, pause'dan menüye.
İleri~80 dk
Proje 3: Game Instance + Multi-slot + Auto Save

BP_GameInstance oluştur, içinde CurrentSaveGame (BP_GameSave ref) ve CurrentSlotName (string) tut. WBP_MainMenu'de 3 slot butonu (Slot1, Slot2, Slot3) — her birinde Does Save Game Exist kontrol et ve "Kayıt var" / "Boş" yaz. Bir slota tıklayınca o slot CurrentSlotName olarak set edilir ve L_Game açılır. Karakter'de her 60 saniyede bir AutoSave (Timer ile) yapılsın — CurrentSlotName'e yaz. Bonus: Save dosyasında player konumu (Vector) saklansın, yüklenince oraya teleport et.

Tamamlama Kontrolü
  • BP_GameInstance Project Settings'te atandı
  • 3 slot butonu farklı kayıtları yönetiyor
  • Slot bilgisi GameInstance üzerinden taşınıyor
  • AutoSave 60 sn'de bir çalışıyor
  • Konum kaydı ve yükleme doğru
  • Bonus: Pause'da manuel Save de aynı slot'a yazıyor
Tam profesyonel save mimarisi — slot yönetimi, auto save, konum dahil tam state.
